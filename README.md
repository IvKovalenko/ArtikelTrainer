# Тренажёр артиклей (der/die/das)

**https://artikel-trainer.pages.dev**

Тренажёр немецких артиклей для бесконечной дрилл-тренировки — без пауз, меню и
лишних кликов. Работает как сайт и ставится на телефон как приложение (PWA).

Проект родился из раздражения существующими тренерами артиклей: одни неудобные,
у других медленные анимации, у третьих слова разбиты по темам — приходится
прыгать между группами и тратить время. Здесь идея другая: открыл — и сидишь
отвечаешь слово за словом, а всё остальное приложение решает само.

Что делает его удобным:

- **Система весов.** Слова, на которые отвечаешь правильно, показываются всё
  реже; слова с ошибками — чаще, пока артикль не запомнится.
- **Новые слова подсовываются постепенно.** Начинаешь с A1; когда уровень
  освоен на ≥95%, открывается следующий (до C2).
- **Никаких пауз.** Ответил → мгновенно увидел результат и перевод → следующее
  слово. С клавиатуры: `1` = der, `2` = die, `3` = das, `4` = Plural,
  `Пробел`/`Enter` — дальше.
- **~2400 слов** — в основном из официальных Wortliste Goethe-Institut
  (A1, A2, B1) плюс B2–C1 лексика из других источников. У каждого слова есть
  русский и английский перевод (показывается тот, что соответствует языку
  интерфейса: русский — при русском, английский — при английском и немецком).
- **Статистика и синхронизация.** Прогресс привязан к аккаунту и общий на всех
  устройствах; в приложении — таблица по каждому слову (верно/ошибки/показы).

Регистрация нужна только чтобы не потерять прогресс: подойдёт **любой
выдуманный email и пароль** (подтверждения по почте пока нет — главное, запомни
их).

## Как это работает

- Статистика хранится одним JSON:
  `{ "Haus": { "correct": 0, "wrong": 0, "seen": 0 }, ... }`.
- Клиент работает **офлайн-first**: пишет в `localStorage`, а в фоне
  синхронизирует с сервером. При старте данные с сервера и локальные
  объединяются поэлементным максимумом (прогресс не теряется).
- Следующий уровень открывается, когда на текущем освоено ≥95% слов (доступны
  только слова из открытых уровней).

### Выбор следующего слова

Слово выбирается **не равномерно**, а взвешенным случайным способом: у каждого слова есть
«вес» — чем он больше, тем выше шанс, что покажут именно это слово. Так тренажёр сам чаще
подсовывает то, что тебе тяжелее даётся, и почти не тратит время на давно выученное.

Вес слова зависит от статистики ответов по нему:

- **новое** слово (ещё ни разу не показывали) — вес **5**: новые слова вводятся охотно;
- **хорошо выученное** (3+ верных ответов подряд и ни одной ошибки) — вес **0.1**: повторяется
  очень редко, только чтобы не забылось;
- **все остальные** — вес считается по формуле `1 + ошибки×3 − верные×0.5` и не опускается
  ниже **0.2**. То есть каждая ошибка резко повышает частоту показа, а каждый верный ответ —
  постепенно понижает.

Как из весов получается конкретное слово (принцип «рулетки»):

1. Берётся пул слов из всех открытых уровней; из него исключается слово, показанное прямо
   перед этим, чтобы одно и то же не выпадало два раза подряд.
2. Складываются веса всех слов пула — получается общая сумма.
3. Выбирается случайная точка от 0 до этой суммы, и слова «проходятся» по очереди, вычитая
   их веса из точки; на каком слове точка ушла в ноль — то и показывается.

Благодаря этому слово с весом 5 выпадает примерно в 25 раз чаще слова с весом 0.2 — но у любого
слова шанс всё равно ненулевой, поэтому подборка не зацикливается на нескольких словах.
